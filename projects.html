TaskFlow – CI/CD to AWS with Observability

Production Ready
Jenkins • Docker • AWS EC2 • Prometheus • Grafana

Problem Statement

Build a production-style delivery pipeline for a containerized API with automated deploys on commit and operational visibility without persistent cloud cost. Required webhook-driven CI/CD, Dockerized builds, cloud deployment, health checks, and metrics while avoiding 24/7 paid instances.

Solution Architecture

GitHub Repo
→
Webhook
→
Jenkins Pipeline
→
Docker Build
→
EC2 Deploy
→
Prometheus/Grafana

Pipeline Highlights

  • Declarative Jenkinsfile with stages for checkout, lint/test, image build, SSH deploy, and automated health verification
  • Repository READMEs document commands and reasoning for full reproducibility
  • Automated rollback capabilities and environment promotion strategies

Observability Implementation

  • Exposed Prometheus metrics in correct text exposition format
  • Verified target status UP in Prometheus with proper scrape configurations
  • Built Grafana dashboard panels for request rate, latency, task counts, and app version
  • Implemented alerting patterns for critical system metrics

Key Obstacles & Resolutions

Webhook 200 but no build: Root cause was incorrect endpoint and branch conditions; fixed with proper /github-webhook/ route and branch handling.
ARM64/AMD64 mismatch: Reprovisioned target to ARM64 Amazon Linux 2023 for architecture compatibility.
SSH/variable scoping: Fixed quoting and environment use for tag/version visibility in remote shell execution.

Results

Deployment Time Reduced from 45min to 8min
Reliability 99.2% deployment success rate
Cost Optimization On-demand provisioning saves 80% cloud costs
Observability Full metrics visibility with alerting

Lessons Learned

Least-privilege webhooks and SCM triggers, clear pipeline observability, and explicit architecture targeting reduce failure modes and speed up recovery. CSRF and reverse proxy patterns recognized as production requirements for security hardening.
